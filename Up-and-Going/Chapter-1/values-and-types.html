Values and Types

Different representations for values are called types in programming terminology. JS has built-in types for each of these so called primitive values

  • For Math: number
  • To print a value on the screen: string (one or more characters, words, sentences)
  • To make a decision in your program: boolean (true or false)

Converting Between Types

If you have a number but need to print it on the screen, you need to convert the value to a string. This is called "coercion".

JS provides facilities for forcibly coercing between types. For example:

var a = "42";

var b = Number( a );

console.log( a ); //42

console.log( b ); //42

see results also in dev console

The string "99.99" and the number 99.99 are the same value but expressed by two different types. One could say they are "loosely equal."

JS will sometimes implicitly coerce values to matching types.

Implicit coercion can and should be learned if you want to take JS programming seriously.
