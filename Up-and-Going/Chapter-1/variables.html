Variables

Variable
A symbolic container assigned and used to hold a value -- so called because the value in this container can vary over time as needed.

Static typing, otherwise known as type enforcement, is typically cited as a benefit for program correctness by preventing unintended value conversions.

Weak typing, otherwise known as dynamic typing, allows a variable to hold any type of value at any time. Cited as a benefit for program flexibility. Variables can hold values of any type without any type enforcement. JS uses this type.

Consider:

var amount = 99.99;

amount = amount * 2;

console.log( amount ); // 199.98


// convert 'amount' to a string, and

// add "$" on the beginning.

amount = "$" + String( amount );

console.log( amount ); //"199.98"

The primary purpose of variables is to manage program state.

In other words, state is tracking the changes to values as your program runs.

Another common usage is for centralizing value setting. This is more typically called constants, when you declare a variable with a value and intend for that value to not change througout the program.

Constants are declared at the top of the program, and by convention, JS variables as constants are usually capitalized, with underscores between words.

Consider:

var TAX_RATE = 0.08; // 8% sales tax


var amount = 99.99;


amount = amount * 2;


amount = amount + (amount * TAX_RATE);


console.log( amount ); // 215.9784

console.log( amount.toFixed( 2 ) ); // "215.98"

The TAX_RATE variable is only constant by convention. There is nothing in the above program that prevents it from being changed.

There is a new way to declare constants. by using const instead of var.

Consider:

// as of ES6 (version of JS)

const TAX_RATE = 0.08;

var amount = 99.99;

Assigning a value to a const prevents the value from being accidently changed later on in the program.
